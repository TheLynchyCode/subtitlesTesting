Track <track> element

This allows for a WebVTT file to be included in the markup when adding subtitles, captions, descriptions, chapters is desirable.
The following attributes are used within the <track> element to execute this functionality.

  • src="" - is the URL of the .vtt file
  • kind="" - is the type of displayed text eg. subtitles, captions, chapters, descriptions
  • label="" - creates an option label (language) in the subtitles controls.
  • srclang="eg. 'en' " - tells the browser what language the subtitles are in.
  • default - tells the browser to try this track before the other options. It is the preferred option.



Audio <audio controls>

the <audio> element is used in the same manner as the <video> element and thus utilises the same attributes as video;
however, it doesn't accept:

  • width="" and height="" - because the is not video content.
  • poster - because there is no visual component.

Attributes for <video> and <audio> elements

  • controls - embeds a HTML video player within the page and provesides start/stop, volume, playback speed etc...control. functionality
  • width="" - Gives the video player a specified width. Note: aspect ratio is maintained.
  • height="" - Gives the video player a specified height.
  • poster="" - Used for a splash screen or advertisement opportunity.
  • preload="" - Used for buffering large files, and can be one of three values:
    1. none - does not buffer the file.
    2. auto - buffers the media file.
    3. metadata - buffers only the metadata of the file.
  • autoplay - causes the video to play automatically upon loading. Not advised to use as people can find this annoying.
  • loop - causes the video to automatically restart when it finishes.
  • muted - causes the media to play muted by default upon the page loading.

The <source> element

Use this element to create compatibility with as many browsers as possible.
Different browsers use different file formats and codecs and so a file/codec runing on one browser may not be supported on another.
Using the <source> element allows for diffrent formats to be included in the markup and lets selected browser to sift through during loading for a format that it supports.
